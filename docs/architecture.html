2. 3단 캐시 계층

① 브라우저 메모리(_mem) 재방문 즉시 · 네트워크 0 ② Vercel CDN (서울 엣지) HIT ~13ms · 함수/워커 미경유 · LRU evict(용량 비공개) ③ Cloudflare Worker KV CDN MISS 시 · ~50ms · evict 없음(long-tail 전부 보관) ④ D1 사전집계 테이블 KV 미스 시 · 1만행 룩업 ~1ms (raw 130만행 스캔 대체)
계층역할한계 / 용도
Vercel CDN서울 엣지에서 즉시 응답(~13ms)LRU eviction → 핫 bounded 키만(통계·대시보드·recent 전국). 수만 개 욱여넣으면 압력으로 핫키까지 evict (실측: 21k 워밍→177초 뒤 91% MISS)
Worker KVCDN 미스 백업 · 응답/raw 캐시evict 없음 → long-tail(지역상세·단지모달 2만개) 전부 보관. Vercel MISS여도 CF→KV ~50ms
D1 사전집계overview/통계/추이 즉시 룩업실시간 집계 금지 → ingest 시 region_month_agg 갱신

3. 캐시 워밍 (사용자가 닿기 전에 데움)

대상키 규모방식
핫 bounded
통계·대시보드·recent 전국		~수백Vercel CDN 워밍 (cron 30 0 * * *, warmCore) — 용량 내라 유지됨
단지 상세 모달2만+Worker KV 워밍 (cron 42 0 * * *, 큐 분산) — CDN 아님(evict 회피)
지역상세·시도탭수천워밍 안 함 → ingest 시 KV 적재 + 프론트 점진 렌더(방지책)
갱신 시점변경분만MOLIT 적재(ensureMonth) 직후 그 지역·단지 키 즉시 워밍
교훈(측정으로 확정): 21,000개 단지모달을 Vercel CDN에 워밍했더니 177초 만에 91%가 MISS(LRU evict + PoP 분산 추정). → long-tail은 CDN이 아니라 evict 없는 Worker KV + 점진 렌더로. CDN 워밍은 핫 bounded에만.

4. 응답 경로별 지연

대시보드/통계(핫) 사용자 → Vercel CDN HIT ~13ms 지역상세/단지모달 사용자 → Vercel CDN MISS → CF워커 KV HIT ~50ms 완전 콜드(최초 1회) 사용자 → Vercel → CF → D1 집계/MOLIT ~0.2~2s (이후 캐시) 모달 점진 렌더 이력=즉시 / 지도·주변시설=비동기 채움 체감 0 (빈 화면 없음)
